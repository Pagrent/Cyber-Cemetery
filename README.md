# 赛博墓园 (Cyber Cemetery)

为数字存在建造一座静默的纪念碑

赛博墓园是一个完全静态的、可无限复制的数字空间。它并非只为悼念逝者，更是为所有希望在浩瀚数据之海中留下自己印记的人而设。无论是永久告别网络、暂时隐退，还是仅仅为了宣告“我曾在此”，你都可以在此创建一座属于自己的数字墓碑。

这里没有社交互动，没有算法推荐，只有纯粹的信息展示。你的墓碑一旦建立，便成为互联网石碑上永恒的一部分，可被任何人镜像、保存和访问。

## 核心精神

* **静默**：无评论，无点赞，只有单向的阅读。
* **自主**：由你完全掌控内容，无需依赖任何平台。
* **永恒**：纯静态文件，脱离服务器亦可存在。
* **平等**：所有墓碑格式统一，彰显数字存在的平等性。

## 项目构成

本项目包含一个静态网站生成器 (`generator.py`) 和一套前端模板（HTML, CSS, JS）。生成器会读取原始墓碑数据，将其编译成一套可直接部署的静态网站文件。

### 输入：墓碑数据 (`/tombstones_raw/`)

每个墓碑由一个独立的 JSON 文件定义，文件名即为其唯一ID。

**JSON 格式示例 (`12345.json`):**

```json
{
  "id": "12345",
  "name": "星尘漫游者",
  "avatar": "https://example.com/avatar.jpg",
  "epitaph": "我选择在此刻下线。\n我的思想已化作数据流，散入虚空。",
  "created": "2024-05-20T08:00:00Z",
  "links": [
    { "url": "https://github.com/stardust", "title": "GitHub" },
    { "url": "https://blog.example.com", "title": "个人博客" }
  ]
}
```

**字段说明:**

* `id` (**必需**): 唯一标识符，必须与文件名（不含`.json`）一致。
* `name` (**必需**): 你的网名或代号。
* `avatar` (**必需**): 头像的公开可访问URL。
* `epitaph` (**必需**): 你的碑文，可以是告别、宣言、诗歌或任何你想留下的文字。
* `created` (**可选**): 创建时间（ISO 8601格式），用于在详情页显示。
* `links` (**可选**): 一个链接数组，每个链接包含 `url` 和 `title`。

### 输出：静态网站 (`/dist/`)

运行生成器后，会输出一个完整的静态网站到 `/dist/` 目录。

**目录结构:**

```
/dist/
├── index.html           # 第一页
├── page_2.html          # 第二页
├── page_3.html          # 第三页
├── ...                  # 更多分页
├── css/styles.css       # 样式表
├── js/main.js           # 交互逻辑
└── assets/
    ├── sun.svg          # 主题切换图标
    └── moon.svg         # 主题切换图标
```

## 网站功能与体验

### 1. 浏览与探索

* **网格布局**: 墓碑以简洁的卡片网格形式呈现，展示头像、名字和ID。
* **分页导航**: 底部提供清晰的分页控件，方便浏览大量墓碑。
* **响应式设计**: 在手机、平板和桌面电脑上均有良好的视觉效果。

### 2. 查看墓碑详情

点击任意墓碑卡片，将弹出一个优雅的模态框（Modal），其中包含：

* 放大的圆形头像。
* 完整的网名、ID和创建日期（如果提供）。
* 完整的碑文，保留原始换行格式。
* 所有外部链接，可直接点击跳转。

### 3. 个性化浏览体验

* **主题切换**: 页面右上角的按钮可在深色/浅色模式间切换，偏好设置会保存在本地。
* **平滑动画**: 页面内容加载时带有淡入效果，弹窗出现也伴有流畅的过渡动画，提升整体质感。

## 核心原则

* **去中心化**: 网站不依赖任何特定服务器或数据库，生成后即可在任何地方托管或离线查看。
* **不可变性**: 一旦生成，墓碑内容即固定。要更新内容，需重新生成整个站点。
* **极简主义**: 没有冗余的装饰、广告或追踪代码，界面只为内容服务。
* **开放性**: 整个项目开源，任何人都可以克隆、修改并创建自己的“墓园”。

## 如何使用

1. **准备数据**: 在 `tombstones_raw` 目录下，按规范创建你的墓碑 JSON 文件。
2. **配置生成器**: 修改 `config.yaml` 文件，设定每页显示的墓碑数量 (`per_page`)。
3. **生成网站**: 运行 `python generator.py` 脚本。
4. **部署或分享**: 将 `dist` 目录中的所有文件上传至 GitHub Pages、Netlify 等静态托管服务，或直接打包分享给他人。
